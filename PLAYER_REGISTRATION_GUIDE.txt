📝 FORM FIELDS EXPLANATION
═══════════════════════════════════════════════════════════════════════════════

1. NAMA PESERTA (Text Input)
   ├─ Label: "📝 Isi Nama Peserta:"
   ├─ Placeholder: "Masukkan nama Anda..."
   ├─ Required: YES (validation check)
   ├─ Length: Any (no limit)
   ├─ Error message: "❌ Silakan masukkan nama Anda terlebih dahulu!"
   └─ Max display: Shows in game header as "📝 Peserta: [Nama]"

2. TIPE DOKTER (Selection Grid)
   ├─ Label: "🏥 Siapakah Kamu? Pilih Tipe Dokter:"
   ├─ Format: 2×2 grid with 4 options
   ├─ Required: YES (validation check)
   ├─ Options:
   │  ├─ 👨‍⚕️ Dokter Klinik (Easy, 30 HP)
   │  ├─ 👩‍⚕️ Dokter Puskesmas (Medium, 20 HP)
   │  ├─ 🚑 Dokter IGD (Hard, 10 HP)
   │  └─ 🏥 Dokter 3T (Very Hard, 5 HP)
   ├─ Visual feedback:
   │  ├─ Default: Border colored (blue/purple/pink/orange)
   │  ├─ On select: Background turns dark blue
   │  └─ Selected text shows: "Pilihan: [Dokter Name]"
   └─ Error message: "❌ Silakan pilih tipe dokter Anda!"

═══════════════════════════════════════════════════════════════════════════════
✅ VALIDATION RULES
═══════════════════════════════════════════════════════════════════════════════

NAMA PESERTA:
├─ Must be filled (not empty)
├─ Trimmed (whitespace removed)
├─ Can contain: Letters, numbers, spaces, special characters
├─ If empty → Alert: "❌ Silakan masukkan nama Anda terlebih dahulu!"
└─ Then: Focus back on name input field

TIPE DOKTER:
├─ Must be selected (one of: klinik, puskesmas, igd, tiga_t)
├─ If not selected → Alert: "❌ Silakan pilih tipe dokter Anda!"
└─ Then: User must click a dokter option

BUTTON STATE:
├─ "⚔️ MULAI PERTARUNGAN!" button always active
├─ Validation happens on click
├─ If validation fails: Show alert, don't proceed
└─ If validation passes: Start game

═══════════════════════════════════════════════════════════════════════════════
🔧 CODE FUNCTIONS
═══════════════════════════════════════════════════════════════════════════════

1. selectClass(classType)
   ├─ Called: When user clicks dokter card (STEP 1)
   ├─ Action: Hide class selection, show registration form
   ├─ Parameters: classType (klinik, puskesmas, igd, tiga_t)
   └─ Result: Registration form displayed

2. selectDokter(dokterType, element)
   ├─ Called: When user clicks dokter option in registration (STEP 2)
   ├─ Action: Highlight selected dokter, show confirmation
   ├─ Parameters: dokterType, HTML element
   └─ Result: Visual feedback + confirmation text

3. startGameWithPlayer()
   ├─ Called: When user clicks "MULAI PERTARUNGAN!" button
   ├─ Validation:
   │  ├─ Check name not empty
   │  └─ Check dokter selected
   ├─ If validation fails: Show alert, stop
   ├─ If validation passes:
   │  ├─ Store player name & dokter type
   │  ├─ Set game stats (HP, difficulty, etc)
   │  ├─ Hide registration form
   │  ├─ Show game screen
   │  └─ Initialize game (load boss 1)
   └─ Result: Game starts with player info
